# 時渡り草子 — 平安篇　ゲーム仕様書 v0.1

> Claude Code 実装用。①歴史学習ゲームの第一作（平安）。
> この仕様は「時代を増やすための最初の型」を作ることを目的とする。

---

## 0. 何を作るか（一行）

**令和の子が平安の邸に迷い込み、ひと季節をそこで暮らす。虫取りや川遊びのかわりに、貝合わせや花摘みや月見で日々を過ごすうちに、いつのまにか平安という時代が体に染みている——絵巻の中で暮らす『ぼくのなつやすみ』。**

---

## 1. 北極星（この5箇条は絶対に外さない）

1. **読ませない。** 画面に長い文章を置かない。解説パネルは作らない。会話は一文ずつ。
2. **体験 → 気づき → さりげない答え合わせ。** 学びは生活に埋め込む。夜、絵日記にそっと一言添えるだけ。
3. **ゲージを見せない。** 数値HUDは出さない。状態は「人の態度・招かれる場・絵日記の充実・情景の変化」で表す（ダイエジェティック＝世界の中で表現する）。
4. **誰でも入れて、深くも学べる。** 操作は「歩く・触れる・選ぶ」だけ。学びの中身は中学発展〜歴史総合＋αを、興味を持った子だけが絵日記から深掘りできる層に格納。
5. **もののあはれ。** 季節の移ろいと、季節の終わりの別れを、情緒の芯に置く。

> 前回プロトタイプの失敗＝(a)文字が多すぎた (b)位・信・雅のゲージを出した (c)時辰を送る紙芝居だった。この3つを全部やめる。

---

## 2. 遊ぶ人と、学びの二層

| 層 | 誰に | 何を | どこで |
|---|---|---|---|
| 表層 | 全員（小学生〜） | 生活の楽しさ。歩く・遊ぶ・人と話す | ゲーム画面そのもの |
| 中層 | 気になった子 | 体験した物事の史実（一言） | 夜の絵日記の一枚ごと |
| 深層 | もっと知りたい子 | 中学発展〜歴史総合＋αの解説 | 絵日記をめくった先（任意） |

遊びは表層だけで完結する。中層・深層に触れなくてもゲームは成立する。「勉強っぽさ」を表に出さないことが最優先。

---

## 3. コアゲームループ

- プレイヤーは平安の**ひと季節**（例：長月〜神無月の秋）を暮らす。実時間で数十分〜数時間、ゲーム内で数週間。
- 1日 = **暁・朝・昼・夕・宵** の時間帯で流れる。急かさない。
- **朝**：目覚め、身支度（装束）。
- **日中**：邸・庭・都の一角を自由に歩き、人と話し、遊ぶ。何をしてもいい。
- **夕**：小さな出来事が一つ起きる（行事・訪問・恋文など）。
- **宵**：その日を絵日記が一枚描く。史実が一言そっと添う。眠る。
- 「ただ過ごす」ことの豊かさを守る。目標達成を急がせない。

---

## 4. 世界（歩ける生活空間）

- **視点**：吹抜屋台（屋根を外して室内を斜め上から見下ろす大和絵の構図）。この俯瞰が本作の視覚的アイデンティティ。
- **空間**：母屋・廂・簀子・対屋・渡殿・南庭・遣水・池・中島・釣殿・門・築地塀・塀の外の小路。
- **季節と時間で変化**：光・空の色・草木・虫の音が移る。秋なら紅葉・菊・すすき・虫の声。
- **表現**：2.5D。絵巻調の背景（AI生成／PD絵巻素材／自作）＋大和絵風キャラ。

---

## 5. 生活の活動（ぼくなつの遊びを平安に翻案）

| ぼくなつ | 平安篇 | 埋め込む学び（表に出さない） |
|---|---|---|
| 虫取り・魚釣り | 庭の草花を摘む・貝を拾う・虫の音を聞く | 季節の景物、自然を愛でる感性＝和歌の源 |
| 草スキー等の遊び | 貝合わせ・偏継ぎ（言葉遊び）・蹴鞠 | 貴族の遊興、仮名・言葉の文化 |
| ラジオ体操・日課 | 書の稽古・琴/笛の稽古・薫物を聞く（香） | 教養（雅）、身分ある者の日々 |
| 家の手伝い | 装束を選ぶ・文（手紙）を届ける | 装束の色目、恋と文のやりとり |
| お祭り | 季節の年中行事（重陽の菊、月見など） | 年中行事と暦、宮廷文化 |
| 絵日記（図鑑） | 集めた草花・貝・和歌・出会った人が絵日記に溜まる | ＝学びの受け皿。ここに史実を格納 |

活動は「やってもやらなくてもいい」。集めたものが絵日記の図譜に溜まっていく（収集の楽しさ）。

---

## 6. 人々（生活する登場人物）

各人が**日課と居場所**を持ち、時間帯で動く（生活感）。話しかけると短い会話。関係が深まると新しい場面が開く。

- **邸の主**（中級貴族・直衣）。昇進を望み、摂関家との縁を気にしている。
- **女房**（十二単）。仮名と和歌の師。プレイヤーに言葉を教える最初の導き手。
- **同い年の姫君 or 若君**。友。ひと季節を通じて関係が育つ主要人物。別れが情緒の核。
- **幼い童**・**老いた従者**。生活に厚みを与える点景。
- **訪ねてくる人**：摂関家の使い、恋文の使者など（外の世界＝政治や恋を運んでくる）。

> 状態（誰と親しいか）は数値で見せない。会話の変化・招かれる場・一緒に過ごせる遊びの増加で表す。

---

## 7. 学びの仕込み方（＝この企画の心臓部）

1. 生活の中に平安の言葉・習慣・物が**解説なしで**自然に現れる。
2. **宵の絵日記**：その日体験したことが一枚の絵になる。絵の隅に手書き風の一言（例：「けふ、貝あはせをした。」）＋ごく小さく史実の栞（例：小さな印＋「貴族の遊び」）。
3. その栞を**めくると**、興味を持った子だけが一段深い解説（中学発展〜歴史総合＋α）に触れられる。めくらなくても遊びは完結。
4. **押し付けの解説モーダルは絶対に出さない。** 「学ばせよう」とする瞬間に没入は死ぬ。

**史実からの逸脱方針**：史実になぞる（if世界線は描かない）。ただし主人公＝令和の子は史実に無い「余白の観察者」なので、歴史の大事件を動かさず、その傍らの生活を生きる。

---

## 8. 状態管理（不可視・ダイエジェティック）

- **可視ゲージ（位・信・雅など）は表示しない。** ← 前回プロトタイプの反省点。
- 裏では最小限の状態を持つ：`親密度（人物ごと）`／`季節・日付の進行`／`体験済みの活動・行事`／`集めた図譜`。
- これらの表現は必ず**世界の中で**：
  - 親しくなる → その人が新しい遊びに誘う、打ち明け話をする、絵日記の登場が増える。
  - 教養が深まる → 歌の会に招かれる、難しい遊びが解禁される。
  - 季節が進む → 草木・光・虫の音・行事が変わる。
- **「体系（平安社会の仕組み）を出口に」の再定義**：信長の野望のような俯瞰ストラテジーUIは作らない。摂関政治・荘園・身分といった「時代の骨組み」は、生活の積み重ねの果てに**気づきとして立ち上がる**（例：「この邸の主がなぜ摂関家に頭を下げるのか、暮らすうちに分かってくる」）。俯瞰は"仕組みが腑に落ちる瞬間"として実装し、管理画面としては実装しない。

---

## 9. 物語の背骨（縦糸・全時代への枠）

先に骨格だけ固定し、中身は時代ごとに実装する。

- **動機**（なぜ時を渡るのか）：※要決定。候補＝現代で失くした何か／古い巻物・鏡・扇に触れて／祖母の家の蔵で。
- **縦糸**：各時代で出会った人の想い・選択が、時を越えて誰かへ受け継がれ、最後に現代の自分自身へつながる。
- **平安の役割**：全体の第一歩＝原点の共感。「千年前の人も、季節を愛で、人を想い、日々を生きていた」を体で知る。
- **季節の終わり = 別れ = 次の時代への扉**。もののあはれ。
- **終着点**：現代に戻ったとき、自分が長い連なりの先端に立っていると気づく。

> 全時代の順（縄文→…→現代）と各時代の役割・縦糸は、平安の型が固まり次第、別途「背骨シート」で確定する。

---

## 10. アートディレクション

- **絵巻美学2.5D**。源氏物語絵巻・大和絵の様式。吹抜屋台の俯瞰。
- **色**：岩絵具（緑青・群青・朱・金泥・胡粉・墨線）。季節で配色が移る。
- **演出**：すやり霞（金雲）で場面転換・時代転換。引目鉤鼻の人物。
- **アセット戦略**：背景・人物はAI画像生成＋自作＋PD絵巻素材。日本特定歴史の3Dアセット調達難を、様式化2.5Dで回避する（これが平安を3Dでなく2.5Dにした理由）。
- **ビルド時静的事前生成**：Max 20xで場面・会話・分岐を事前生成して焼き込み、実行時はAPIを呼ばない → 無料運用・オフライン動作可能。

---

## 11. UI/UX（小学生でも／文字最小）

- **操作は3つだけ**：歩く（タップ移動）／触れる（人・物）／選ぶ（短い選択肢）。
- 画面に長文を置かない。会話は一文ずつ、絵とともに。
- **メニュー最小**：絵日記／季節・時間（絵で表示）／持ち物（草花・貝の図譜）。それ以上増やさない。
- **迷わせない**：今できることが絵で分かる（触れられる物はほのかに際立つ）。
- チュートリアルも読ませない：最初の朝、女房に導かれて自然に操作を覚える。

---

## 12. 技術アーキテクチャ（Claude Code 向け）

**3層構造**（③十五少年漂流記と型を共通化する前提）

- コンテンツデータ（時代ごとに差し替え）：平安データセット＝空間・間取り／人物と日課／活動／イベント／絵日記の図譜と史実／季節の景物／アート
- ジャンルモジュール（没入生活ゲーム共通）：生活ループ／時間・季節／移動／会話／絵日記／親密度／ダイエジェティック状態
- コアエンジン：レンダリング(2.5D)／入力／セーブ／シーン管理／アセットロード

- **技術**：React Three Fiber + Zustand + Vite + PWA（③と共通）。
  - 平安は**2.5D**：板ポリゴン＋絵巻テクスチャ、**正投影（オルソ）に近い俯瞰カメラ**で吹抜屋台を表現。
  - 代替案：軽量2D（React + Canvas / PixiJS）。**エンジン共通化（R3F）か平安単体の軽さかは実装初日に判断**。推奨＝まず2.5DをR3Fで薄く試作し、重ければ2Dへ退避。
- **状態**：Zustand。不可視状態を保持し、ダイエジェティックに反映。
- **データ駆動**：時代＝差し替え可能なデータセット。共通エンジンは時代非依存。
- **デプロイ**：Vercel / GitHub Pages、PWA、オフライン可。
- Godot/Unity は使わない。

---

## 13. 時代を増やす「型」

| 不変（共通エンジン＋ジャンル） | 時代ごとに差し替え（データ） |
|---|---|
| 生活ループ・時間・季節 | 空間／間取り・背景アート |
| 移動・会話・選択 | 人物と日課 |
| 絵日記システム | 活動セット（遊び） |
| 親密度・不可視状態 | イベント／行事 |
| UI・操作 | 絵日記の図譜＋史実データ |
| セーブ・シーン管理 | 季節の景物・配色 |

---

## 14. MVP（最初に作る縦スライス）

**平安の「一日」を、ぼくなつ的に暮らせる**ところまで。

1. 朝、目覚めて身支度（装束を選ぶ）。女房に導かれて操作を覚える。
2. 邸と庭を**自由に歩く**。触れられる物・人がほのかに際立つ。
3. **3人と短い会話**（主・女房・同い年の友）。会話は一文ずつ。
4. **1つの遊び**を体験（貝合わせ or 花摘み）。集めたものが図譜に入る。
5. **夕方に小さな出来事**が一つ（例：友から紅葉を添えた文が届く）。
6. **宵に絵日記が一枚**描かれ、史実が一言そっと添う。栞をめくると深掘りできる。眠る。

**達成条件**：ゲージを一切見せず、長文を一切出さず、「読まずに平安の一日を暮らした」感覚が生まれること。

---

## 15. 前回プロトタイプからの変更点

- 詞書・解説カードの長文を**全廃**。会話は一文ずつ、解説は絵日記の任意層へ。
- 可視ゲージ（位・信・雅）を**廃止** → ダイエジェティック表現へ。
- 時辰を送る紙芝居 → **自由に過ごす生活ループ**へ。
- 「信長の野望的な俯瞰ストラテジーUI」→ **作らない**。
